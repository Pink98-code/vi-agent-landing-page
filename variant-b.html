The briefing

One system, read at any altitude.

The operating value is legible without a technical background, and the system boundaries hold up to close inspection.

What changes on the floor

Fragile handoffs go away. Files go in, the system shows exactly where they were routed, run health stays visible, risky changes wait for a human yes, and every session ends with concrete deliverables.

  • Clear status for every client and pipeline.
  • Decision gates keep human judgment in the loop.
  • Analytics separate internal ops from partner-facing proof.

Where it compounds

The wedge is operational AI with compliance-aware workflow design: deterministic processing, agent reasoning only where judgment is valuable, and telemetry from run health, approvals, and deliverable history.

  • Expands from a local cockpit toward managed enterprise workflow.
  • Compounds with client history and cross-client patterns.
  • Anchored in painful, recurring pharma data operations.

How it actually works

A React/Vite console talks to a Fastify control server, a SQLite run registry, live event streams, typed analytics endpoints, MCP tools, Python analysis, and track-specific agent runners whose output is critiqued before it ships.

  • The run registry is the shared source of truth.
  • Typed MCP tools expose parse, validate, QA, and build.
  • Each track owns its own inputs, outputs, and rules.

The roster

Judgment where it changes the outcome.

Reasoning is spent only where it moves the result. Everything that can be deterministic stays deterministic, typed, and reproducible.

Reasoning agents

  • Orchestrator: answers from the system's own data and dispatches runs only after confirmation.
  • Audit analyst: reasons over pre-parsed audit context and proposes optimizations.
  • Search Insights builder: turns a brief or profile into a five-sheet workbook.
  • Claims QA: reconciles a JSON audience config against the CSV source-of-truth.
  • Client-brief parser: extracts brand, codes, audiences, indicators, and KOLs.
  • Upload classifier: resolves track, client, and audience from messy uploads.
  • Client classifier: places each client in a seven-category taxonomy.
  • Agent-text QA: scores agent text against eight standards.
  • Validation critic: drives the bounded self-correction loop.
  • Knowledge curator: prunes patterns and proposes standing-rule changes.
  • Conflict detector: flags docs and rules that no longer match the code.
  • Decision capture: distills operator decisions into institutional memory.

Deterministic engines

  • Parse → validate → finalize: agents think only in the middle.
  • Quality gate: one check for consistency and completeness.
  • Workbook builders: four- and five-sheet templates, reproducible.
  • Schema validators: shared typed contracts that fail loud.
  • Pattern extractor: turns runs and corrections into reusable patterns.
  • Analysis engine: trends, anomalies, clustering, prediction, similarity.
  • Dispatcher & watchdog: advances stages, fans out runs, recovers hung work.
  • Upload router: classify, dedupe by hash, hard-link into the input tree.
  • MCP server & dashboard: typed, test-backed tools plus a health surface.
