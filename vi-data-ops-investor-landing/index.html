Where it compounds

The wedge is operational AI with compliance-aware workflow design: deterministic processing, agent reasoning only where judgment is valuable, and telemetry from run health, approvals, audience policy gates, and deliverable history.

  • Expands from a local cockpit toward managed enterprise workflow.
  • Compounds with client history and cross-client patterns.
  • Anchored in painful, recurring pharma data operations.

How it actually works

A React/Vite console talks to a Fastify control server, a SQLite run registry, live event streams, Zod-backed OpenAPI, typed analytics endpoints, MCP tools, Python analysis, and track-specific agent runners whose output is critiqued and corrected before it ships.

  • The run registry is the shared source of truth.
  • Typed MCP tools expose the parse, validate, QA, build, and knowledge actions.
  • Each track owns its own inputs, outputs, and validation rules.
  • Multi-audience fan-out is tracked with a shared run_group_id.

A live multi-agent system, one source of truth

Every path here is a real boundary in the codebase — the run registry sits at the center, deterministic mechanics stay reproducible, and judgment-heavy work is fenced inside agents that critique and correct each other before anything is delivered.

What holds up under close inspection

  • A shared registry records every run, stage, gate, decision, log, and artifact; an append-only event log replays it.
  • Reasoning agents are isolated per track; each maker’s output passes a validation critic, with a bounded self-correction loop on failure.
  • A consolidated quality gate checks cross-reader consistency and artifact completeness before a run can finish.
  • Model tier is resolved from the auth context — heavier reasoning where judgment pays, cheap classifiers where it does not.
  • Deterministic tools are typed and test-backed: parse, validate, finalize, QA, build, knowledge.
  • Live events keep the cockpit honest — real stage transitions, never invented progress.
  • Zod-backed OpenAPI and shared enums keep UI, server, and tests aligned.
  • Multi-user access with roles and a one-switch private mode, and authenticated deliverable downloads.
  • Corrections and cross-client patterns compound into a registry that future runs read from.

Judgment where it changes the outcome; mechanics everywhere else.

Reasoning is spent only where it moves the result. Everything that can be deterministic stays deterministic, typed, and reproducible — the system is auditable, not a black box.

Reasoning agents

  • Orchestrator — answers from the system’s own data and dispatches runs only after an explicit confirmation.
  • Audit analyst — reasons over pre-parsed audit context and proposes optimizations.
  • Search Insights builder — turns a brief or profile into a five-sheet activation workbook.
  • Claims QA — reconciles a JSON audience config against the CSV source-of-truth.
  • Client-brief parser — extracts brand, codes, audiences, indicators, and KOLs from a brief.
  • Upload classifier — resolves track, client, and audience from messy uploads on a lightweight tier.
  • Client classifier — places each client in a seven-category taxonomy from weighted signals, pinned by golden fixtures.
  • Agent-text QA — scores agent text against eight standards before it reaches a partner.
  • Validation critic — reviews every maker’s output and drives the bounded self-correction loop.
  • Knowledge curator — prunes stale patterns, calibrates, and proposes standing-rule changes for review.
  • Conflict detector — flags documentation and rules that no longer match the running code.
  • Decision capture — distills operator gate decisions into reusable institutional memory.

Deterministic engines

  • Parse → validate → finalize — the optimization pipeline; agents think only in the middle.
  • Quality gate — one check for cross-reader consistency and artifact completeness before a run can finish.
  • Workbook builders — the four- and five-sheet Excel templates, reproducible byte-for-byte.
  • Schema validators — shared typed contracts that fail loud instead of drifting.
  • Pattern extractor — the learning sink that turns runs and corrections into reusable patterns.
  • Analysis engine — trends, anomalies, clustering, prediction, and cross-client similarity.
  • Dispatcher & watchdog — advances stages, fans out grouped runs, and recovers hung work.
  • Upload router — classify, dedupe by content hash, and hard-link into the canonical input tree.
  • MCP server & dashboard — typed, test-backed tools plus a separate embedded health surface.
